# เอกสารข้อกำหนดความต้องการซอฟต์แวร์ (SRS)
## ระบบบริหารจัดการร้านเช่าเสื้อผ้า
### เวอร์ชัน 1.0

## 1. บทนำ
### 1.1 วัตถุประสงค์
เอกสารนี้อธิบายข้อกำหนดความต้องการซอฟต์แวร์สำหรับระบบบริหารจัดการร้านเช่าเสื้อผ้า โดยมีวัตถุประสงค์เพื่อ
- ควบคุมการคืนและตรวจสอบสภาพเสื้อผ้า
- จัดการค่าเช่า ค่าปรับ และค่ามัดจำ
- ติดตามสถานะการเช่าและตารางการจอง
- จัดการลูกค้าและสมาชิก
- สร้างรายงานและวิเคราะห์ข้อมูล  
- เพื่อสร้างและนำแพลตฟอร์มการเช่าเสื้อผ้าออนไลน์มาใช้เพื่อให้การดำเนินการของร้านมีประสิทธิภาพ   สูงสุด  
- เพื่ออำนวยความสะดวกให้ลูกค้าสามารถเข้าถึงค้นหาและการเช่าเสื้อผ้าได้อย่างรวดเร็ว  
- เพื่อปรับปรุงและเพิ่มประสิทธิภาพในการจองและสถานการณ์เช่า  
- เพื่อขยายช่องทางการตลาดและเข้าถึงกลุ่มลูกค้าใหม่ๆผ่านแพลตฟอร์มออนไลน์  
- เพื่อให้ลูกค้าสามารถตรวจสอบสถานะการจองการชำระเงินและกำหนดส่งคืนได้ตลอดเวลา


### 1.2 ขอบเขต
ระบบจะครอบคลุมการทำงานหลักดังนี้
- การจัดการเสื้อผ้า/ชุด
- การจองและเช่า
- การคืนและตรวจสอบสภาพ
- การจัดการลูกค้า
- การคำนวณค่าเช่าและค่าปรับ
- การจัดการมัดจำ
- การจัดการตารางการเช่า
- การออกรายงาน   
- ระบบจัดการข้อมูลของเสื้อผ้าแต่ละหมวดหมู่

### 1.3 ประเภทร้านที่รองรับ

  # เอกสารขอบเขตของงาน 
## ข้อมูลโครงการและวัตถุประสงค์ 
เอกสารนี้เป็นข้อกำหนดขอบเขตของงานสำหรับระบบบริหารจัดการร้านเช่าเสื้อผ้าออนไลน์ โดยมีวัตถุประสงค์หลักเพื่อพัฒนาระบบที่จัดการงานเช่าเสื้อผ้าหน้าร้านและออนไลน์
มุ่งเน้นการเพิ่มประสิทธิภาพในการดำเนินการของร้านการอำนวยความสะดวกในการเข้าถึง
และจองชุดให้ลูกค้า และการขยายช่องทางการตลาด

## ขอบเขตของงาน
ขอบเขตของระบบที่พัฒนาขึ้นจะครอบคลุมฟังก์ชัน
- ผู้ใช้สามารถสั่งซื้อ/จองได้ตลอดเวลา
- ระบบแจ้งเตือนตามเวลาที่ตั้งไว้
- ผู้ใช้ต้องสแกน QR Code เพื่อเปิดการแจ้งเตือน 
- ระบบบันทึกประวัติการดำเนินการ 

## สิ่งที่อยู่นอกขอบเขตงาน 
สิ่งที่อยู่นอกเหนือขอบเขตงานและจะไม่ถูกพัฒนาในโครงการนี้
- ไม่ใช้ระบบปัญญาประดิษฐ์ (AI) วิเคราะห์พฤติกรรม

## ฟังก์ชันหลักของระบบ
ฟังก์ชันหลักที่รวมอยู่ในระบบประกอบด้วย
- ระบบสมัครสมาชิกและเข้าสู่ระบบสำหรับลูกค้าและเจ้าหน้าที่
- ระบบตั้งเวลาและแจ้งเตือน การแจ้งเตือนก่อนรับ/คืนชุด
- ระบบสแกน QR Code 
- ระบบแสดงประวัติการใช้งาน 

## ระยะเวลาดำเนินงาน 
ระยะเวลาโดยประมาณสำหรับการดำเนินงานแบ่งเป็น 4 สัปดาห์หลัก
- สัปดาห์ที่ 1: วิเคราะห์และออกแบบระบบ ทบทวน SRS, ออกแบบ UX/UI, ออกแบบฐานข้อมูล
- สัปดาห์ที่ 2-3: พัฒนาระบบ Development - พัฒนา Back-end, Front-end, เชื่อมต่อ Payment Gateway
- สัปดาห์ที่ 4: ทดสอบและปรับปรุงระบบ
- สัปดาห์ที่ 5: จัดทำรายงานและนำเสนอ

## ผลลัพธ์ที่ต้องส่งมอบ
ผลลัพธ์ที่ต้องส่งมอบให้กับลูกค้า
- Source Code ของระบบ
- รายงานโครงการ  
- เอกสารคู่มือการใช้งานและการติดตั้ง
- การสาธิตการทำงานของระบบ Version Beta
